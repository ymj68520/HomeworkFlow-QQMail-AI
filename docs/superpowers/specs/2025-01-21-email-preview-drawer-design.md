# 邮件预览侧边栏设计文档

**日期**: 2025-01-21
**状态**: 已批准
**作者**: Claude

## 概述

在QQ邮箱作业收发系统的主窗口中添加可滑动的侧边抽屉（Drawer），用户双击表格中任何邮件条目时，侧边栏从右侧滑入显示该邮件的完整信息。支持智能切换、固定模式和完整的附件操作功能。

## 设计目标

1. **快速预览** - 无需下载完整邮件内容即可查看所有已收集信息
2. **非侵入式** - 侧边栏不影响主窗口操作，可随时关闭
3. **智能切换** - 浏览不同邮件时内容平滑更新，提升效率
4. **完整操作** - 支持附件的文件级操作（打开、定位、重命名等）

## 组件架构

### 新增组件

**EmailPreviewDrawer** (`gui/email_preview_drawer.py`)
- 独立的侧边栏组件类
- 继承自 `ctk.CTkFrame`
- 负责内容渲染、动画控制、用户交互

### 修改组件

**MainWindow** (`gui/main_window.py`)
- 集成侧边栏实例
- 添加双击事件绑定
- 管理侧边栏显示/隐藏逻辑

## 详细设计

### 1. 侧边栏容器

**尺寸规格**
- 宽度：屏幕宽度的30%
- 最小宽度：400px
- 最大宽度：800px
- 高度：100%主窗口高度

**位置与布局**
- 锚定位置：主窗口右侧
- 层级：覆盖在表格上方（z-index提升）
- 展开：从右向左滑入
- 收起：从左向右滑出

**动画参数**
- 滑入/滑出时间：300ms
- 缓动函数：ease-in-out
- 内容切换：150ms淡入淡出

**控制按钮**
- 固定/取消固定按钮（图钉图标）
- 关闭按钮（×图标）
- 位置：侧边栏顶部右上角

### 2. 内容布局 - 卡片式

采用垂直堆叠的4个信息卡片，每个卡片有独立的标题和内容区域。

#### 卡片1: 学生信息

**字段**
- 学号（大号字体）
- 姓名
- 邮箱地址
- 提交状态标签组：
  - 逾期/正常（颜色标识）
  - 已下载 ✓
  - 已回复 ✓

**布局**
- 2列网格
- 学号和姓名突出显示

#### 卡片2: 邮件信息

**字段**
- 邮件主题（大标题，可换行）
- 发件人（email_from）
- 收件时间（received_time）
- 提交时间（submission_time）
- 邮件UID（小号灰色字体）

**布局**
- 主题单独一行
- 其他字段键值对排列

#### 卡片3: 作业信息

**字段**
- 作业名称（assignment_name）
- 本地存储路径（local_path）- 可选择复制
- 数据库记录ID

**布局**
- 作业名称突出显示
- 路径显示为可点击的标签

#### 卡片4: 附件列表

**显示形式**
- 表格布局：文件名 | 大小 | 操作
- 支持多行显示
- 空状态提示："无附件"

**附件操作按钮**
每个附件行包含：
- 打开文件（📄图标）- 用系统默认程序打开
- 打开文件夹（📁图标）- 在文件管理器中定位
- 复制路径（📋图标）- 复制完整文件路径
- 重命名（✏️图标）- 弹窗输入新名称

**操作确认**
- 打开操作：直接执行
- 重命名操作：弹窗确认，显示原文件名和新文件名输入框

### 3. 交互行为

#### 打开侧边栏
**触发**: 用户双击表格中的邮件条目

**流程**:
1. 检测双击事件（`<Double-1>`绑定）
2. 获取点击条目的完整数据（从`self.filtered_submissions`）
3. 调用`drawer.show(submission_data)`
4. 侧边栏滑入动画（如果未显示）或内容切换动画（如果已显示）

#### 智能切换
**场景**: 用户在已打开侧边栏的情况下，双击另一个条目

**行为**:
1. 侧边栏保持打开状态
2. 内容区域淡出旧数据（150ms）
3. 更新为新条目的数据
4. 内容区域淡入新数据（150ms）
5. 更新标题栏显示当前条目的学号和姓名

#### 固定模式
**触发**: 用户点击"固定"按钮

**行为**:
- 按钮状态切换为"已固定"（视觉反馈：按钮高亮或图标变化）
- 侧边栏不再响应自动关闭信号
- 双击其他条目时仍然切换内容
- 点击主窗口其他区域不会关闭侧边栏
- 用户必须手动点击"取消固定"或"关闭"按钮

#### 关闭侧边栏
**触发方式**:
1. 点击关闭按钮
2. 点击主窗口其他区域（仅在未固定时）
3. 按ESC键

**行为**:
- 侧边栏向右滑出（300ms）
- 完全隐藏后释放资源

### 4. 数据流

```
用户双击表格条目
  ↓

如果侧边栏已显示：
  - 播放内容切换动画
  - 更新所有字段数据
  - 更新标题栏
```

### 5. 文件操作实现

#### 打开文件
```python
import os
import subprocess
import platform

def open_file(file_path: str):
    """跨平台打开文件"""
    if platform.system() == 'Windows':
        os.startfile(file_path)
    elif platform.system() == 'Darwin':  # macOS
        subprocess.call(['open', file_path])
    else:  # Linux
        subprocess.call(['xdg-open', file_path])
```

#### 打开文件夹
```python
def open_folder(file_path: str):
    """在文件管理器中定位文件"""
    import subprocess
    import platform

    if platform.system() == 'Windows':
        subprocess.call(['explorer', '/select,', file_path])
    elif platform.system() == 'Darwin':  # macOS
        subprocess.call(['open', '-R', file_path])
    else:  # Linux
        subprocess.call(['nautilus', file_path])
```

#### 重命名文件
```python
def rename_file(old_path: str, new_name: str) -> bool:
    """重命名文件"""
    try:
        import shutil
        directory = os.path.dirname(old_path)
        new_path = os.path.join(directory, new_name)
        shutil.move(old_path, new_path)
        return True
    except Exception as e:
        print(f"重命名失败: {e}")
        return False
```

### 6. 错误处理

**数据缺失**
- 字段值为None或空字符串时显示"未设置"
- 本地路径不存在时显示"文件已删除"并禁用相关操作

**文件操作失败**
- 文件不存在：显示警告对话框，提示文件已被移动或删除
- 权限不足：显示错误对话框，提示需要管理员权限
- 重命名冲突：显示错误对话框，提示目标文件名已存在

**连接错误**
- 侧边栏使用已加载的本地数据，不涉及网络连接
- 所有操作均在本地执行

**字体**

**间距**
- 卡片间距：15px
- 卡片内边距：12px
- 字段间距：8px
