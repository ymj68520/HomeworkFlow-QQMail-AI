# UI 复选框和批量操作功能设计

**日期:** 2026-04-21
**状态:** 已批准

## 概述

为 QQ 邮箱作业收发系统的 GUI 界面添加可工作的复选框列和批量操作功能（批量下载、批量回复、批量删除）。

## 问题分析

### 当前问题

1. **复选框列无效：**
   - "选择"列显示 ☐ 符号，但点击无响应
   - 全选/清除选择按钮无法正确更新复选框状态
   - 用户无法直观地看到哪些项被选中

2. **批量操作未实现：**
   - `on_batch_download()`、`on_batch_reply()`、`on_batch_delete()` 只显示占位提示
   - 批量操作与选中项之间没有联动

### 根本原因

- `ttk.Treeview` 不支持原生复选框控件
- 当前实现只是在列中显示文本符号，没有状态管理和点击事件处理

## 设计方案

### 1. 复选框列实现

**技术方案：使用 Treeview 图像模拟复选框**

- 创建两个 PIL/Pillow 图像对象：`unchecked` (☐) 和 `checked` (☑)
- 在 Treeview 的"选择"列显示这些图像
- 点击"选择"列时切换图像并更新内部状态
- 维护一个字典 `self.checked_items` 追踪每行的选中状态

**数据结构：**
```python
self.checked_items = {
    item_id: True/False  # item_id 是 Treeview 的 item ID
}
```

**事件处理：**
```python
def on_tree_click(self, event):
    """处理 Treeview 点击事件"""
    # 识别点击的列和区域
    # 如果点击的是"选择"列，切换复选框状态
    # 更新图像和选中项列表
```

### 2. 全选和清除选择

**全选按钮：**
- 将所有行的复选框设置为选中状态（☑）
- 更新 `self.checked_items` 字典
- 更新"已选择"标签显示实际选中数量

**清除选择按钮：**
- 将所有行的复选框设置为未选中状态（☐）
- 清空 `self.checked_items` 字典
- 更新"已选择"标签为 0

### 3. 批量操作功能

#### 3.1 批量下载

**行为：**
- 重新下载所有选中记录的附件文件
- 如果本地文件已存在，覆盖原文件
- 更新数据库中的 `local_path` 和 `is_downloaded` 状态

**实现步骤：**
1. 获取所有选中项的 submission_id
2. 从数据库查询这些记录的邮件 UID
3. 调用 `storage_manager.store_submission()` 重新保存附件
4. 更新数据库记录
5. 刷新表格显示

**错误处理：**
- 记录下载失败的项
- 全部完成后显示成功/失败统计
- 不中断批量操作，继续处理下一项

#### 3.2 批量回复

**行为：**
- 只给选中记录中**未回复过**（`is_replied=False`）的记录发送确认邮件
- 使用邮件主题中提取的学生姓名
- 成功发送后更新数据库的 `is_replied` 状态

**实现步骤：**
1. 获取所有选中项
2. 过滤出 `is_replied=False` 的记录
3. 对每条记录调用 `smtp_client.send_reply()`
4. 成功后调用 `db.mark_replied(submission_id)`
5. 刷新表格显示

**错误处理：**
- 记录发送失败的项
- 显示成功/失败统计
- 不重试失败的项（避免重复发送）

#### 3.3 批量删除

**行为：**
- 删除前显示确认对话框
- 列出要删除的记录数量和学生信息
- 用户确认后执行删除
- 同时删除本地文件和数据库记录

**实现步骤：**
1. 获取所有选中项的详细信息
2. 显示确认对话框：
   ```
   确认删除以下 3 条记录？
   - 2021001 - 张三 - 作业1
   - 2021002 - 李四 - 作业2
   - 2021003 - 王五 - 作业3

   此操作不可撤销！
   ```
3. 用户确认后：
   - 删除本地附件文件
   - 从数据库删除记录
   - 删除附件记录
4. 刷新表格和统计数据

**错误处理：**
- 如果文件删除失败，仍删除数据库记录（保持一致性）
- 记录删除失败的项
- 显示删除结果统计

## 风险和注意事项

1. **Treeview 点击事件：**
   - 需要精确计算点击位置判断是否在"选择"列
   - 可能需要处理不同 DPI 缩放

2. **批量操作性能：**
   - 大量记录时可能耗时较长
   - 考虑添加进度条或取消功能

3. **数据一致性：**
   - 批量删除失败时保持数据库和文件系统一致
   - 网络错误时避免部分更新
