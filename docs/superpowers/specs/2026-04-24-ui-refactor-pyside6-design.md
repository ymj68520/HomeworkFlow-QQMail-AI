# UI 重构 (PySide6) 设计规格说明

> **目标**：将现有的基于 `customtkinter` (Tkinter) 的界面重构为现代化的 PySide6 桌面应用。
> **视觉风格**：方案 A - 现代 SaaS 控制面板风格（深色模式，卡片式布局，圆角，阴影）。

## 1. 架构设计

重构将遵循“组件化”和“逻辑与界面分离”的原则。

### 1.1 目录结构
- `gui/`
    - `main_window.py`: 主窗口类，负责组件组装和顶层信号协调。
    - `components/`: 存放原子组件和功能块。
        - `sidebar.py`: 左侧侧边栏（包含过滤和统计）。
        - `data_table.py`: 现代化表格（继承自 `QTreeView` 或 `QTableWidget`）。
        - `drawer.py`: 右侧平滑滑入的详情预览抽屉。
        - `batch_popup.py`: 批量编辑浮动层。
        - `common/`: 存放通用的圆角按钮、标签等。
    - `styles/`
        - `theme.qss`: 核心样式表文件。
        - `palette.py`: 颜色、字体、内边距常量定义。
    - `assets/`: 图标和图像资源。

### 1.2 通信机制
- 使用 Qt 的 `Signal` 和 `Slot` 进行组件间通信。
- 业务逻辑（如数据库操作、IMAP 下载）保持在原有目录下，通过主窗口或 Controller 调用。

## 2. 视觉规范

### 2.1 调色板 (Dark Theme)
- **Background**: `#121212`
- **Surface/Card**: `#1E1E1E`
- **Border**: `#2C2E33`
- **Primary**: `#339AF0` (蓝色)
- **Success**: `#51CF66` (绿色)
- **Error**: `#FF6B6B` (红色)
- **Text Primary**: `#E9ECEF`
- **Text Secondary**: `#ADB5BD`

### 2.2 交互设计
- **Hover 效果**: 所有可点击项在鼠标悬停时应有背景色淡入或轻微放大的效果。
- **滑入动画**: 预览抽屉使用 `QPropertyAnimation` 对其 `geometry` 或 `pos` 进行动画处理。
- **Badge 样式**: 状态列（如“已完成”）使用带圆角背景色的 Badge 标签。

## 3. 功能模块详情

### 3.1 左侧侧边栏 (Sidebar)
- 顶部放置“系统统计”卡片。
- 中部放置折叠式过滤器。
- 底部放置当前连接状态。

### 3.2 数据表格 (DataTable)
- 支持多选。
- 列宽自适应。
- 双击行触发预览抽屉。
- 右键行触发批量修改弹窗。

### 3.3 预览抽屉 (Drawer)
- 滑入式设计，宽度固定为窗口的 30%（或可拖拽）。
- 包含详情区、正文显示区、附件管理区。

## 4. 实现路径

1.  搭建基础目录结构和 QSS 加载机制。
2.  开发并测试单个 UI 原子组件（按钮、输入框、Badge）。
3.  构建侧边栏和数据表格原型。
4.  实现抽屉滑入动画。
5.  对接原有业务逻辑（数据加载、批量操作、邮件下载）。
6.  全局视觉打磨。
