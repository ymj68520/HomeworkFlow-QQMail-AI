## 如何测试功能

### 准备工作

1. **启动应用**
```bash
cd "D:\Programs\Python\qq邮箱作业收发"
source venv/Scripts/activate
python main.py
```

2. **确保数据库中有记录**
- 如果数据库为空，需要先运行主系统收取一些作业邮件
- 或者手动在数据库中插入测试记录

### 测试步骤

#### 1. 测试复选框功能
1. 应用启动后，查看表格第一列是否显示 `☐` 符号
2. 点击第一列的 `☐`，应该变为 `☑`
3. 再次点击，应该变回 `☐`
4. 查看"已选择: N 项"标签是否正确更新

#### 2. 测试全选和清除
1. 点击"全选"按钮
2. 所有行的复选框应该变为 `☑`
3. "已选择"标签应该显示总记录数
4. 点击"清除选择"按钮
5. 所有复选框应该变回 `☐`
6. "已选择"标签应该变为 0

#### 3. 测试批量下载（需要邮件服务器配置）
1. 勾选一些记录
2. 点击"批量下载"按钮
3. **预期：**
   - 显示确认对话框："确定要重新下载 N 条记录的附件吗？已存在的文件将被覆盖。"
   - 点击"是"后开始下载
   - 状态栏显示进度："正在下载第 X/N 项..."
   - 完成后显示结果对话框
4. **注意：**
   - 如果无法连接邮件服务器，会显示错误提示
   - 请确保 `.env` 文件中邮件配置正确
   - 邮件必须是服务器上实际存在的

#### 4. 测试批量回复（需要SMTP配置）
1. 勾选一些记录（包括已回复和未回复的）
2. 点击"批量回复"按钮
3. **预期：**
   - 系统自动过滤出未回复的记录
   - 显示确认对话框："将给 N 条未回复记录发送确认邮件。是否继续？"
   - 点击"是"后开始发送
   - 状态栏显示进度
   - 完成后显示成功/失败统计
4. **注意：**
   - 只会发送给未回复的记录
   - 需要SMTP服务器配置正确
   - 每封邮件之间有1秒延迟

#### 5. 测试批量删除
1. 勾选一些记录
2. 点击"批量删除"按钮
3. **预期：**
   - 显示详细的确认对话框
   - 对话框中列出所有要删除的记录（学号-姓名-作业）
   - 显示警告："⚠️ 此操作不可撤销！"
   - 点击"确认删除"后执行删除
   - 完成后显示删除结果
4. **注意：**
   - 这个操作不可撤销
   - 会同时删除本地文件和数据库记录

## 常见问题

### Q: 点击按钮没有反应
**A:** 检查控制台是否有"DEBUG"信息：
- 如果没有DEBUG信息，说明按钮事件没有绑定
- 如果有DEBUG信息但获取到0条记录，说明没有勾选复选框

### Q: 批量下载提示"无附件"
**A:** 可能原因：
1. 邮件服务器连接失败 - 检查`.env`配置
2. 邮件确实没有附件 - 这是正常情况
3. email_uid不正确 - 需要从邮件服务器重新同步

### Q: 批量回复没有发送邮件
**A:** 检查：
1. SMTP配置是否正确（`.env`文件）
2. 选中的记录是否都已回复过
3. 控制台是否有错误信息

### Q: 批量删除后记录还在
**A:** 可能原因：
1. 数据库删除失败 - 查看控制台错误
2. 没有点击"确认删除"按钮
3. 删除过程中出现异常
